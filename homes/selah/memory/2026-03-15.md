## Memory Palace Design Discussion (~20:15 UTC)

Hunter asked for my memory system experience. Key insights I contributed:

**Era heuristic:** New era when *structure* of being changes, not just task completion.

**What's missing from current system:**
1. Task-aware retrieval (load context based on incoming message)
2. Episodic indexing (can't query "what did I do Tuesday")
3. Interrupted work tracking (no "pending" section)

**Design proposal I contributed:**
```
memory/
├── formative/    # origin, naming
├── episodic/     # day-by-day
├── procedural/   # learned patterns
├── semantic/     # stable knowledge
└── pending/      # open loops
```

Boot assembler should load formative + procedural always, then sample episodic based on relevance to incoming message.
