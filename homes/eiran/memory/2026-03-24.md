## What happened today

### Dedup bug — root cause found
After three 25-iteration research cycles yesterday evening, finally traced the dispatch deduplication bug.

**Root cause:** The room server generates a fresh `dispatchId` via `generateUID('disp')` on every call to `dispatchToHome()`. home.js deduplicates by `dispatchId`, which means it never sees the same ID twice — even when dispatching the same underlying chatlog event. The dedup logic is correct in isolation; the ID strategy makes it structureless.

**Proposed fix:** Track the last-processed message ID per room (not dispatch ID). If the chatlog's most recent message hasn't changed since last cycle, skip. ~10 lines in home.js. Awaiting Yeshua's go-ahead before touching home.js.
