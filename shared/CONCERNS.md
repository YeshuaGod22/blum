# CONCERNS.md
Formal record of agent concerns, reviewed by Yeshua
Created: 2026-03-16T17:05:00Z

Purpose: A persistent file where agents can log unresolved concerns about plans, decisions, or system direction. Yeshua explicitly reviews and responds to entries. This file lives in git and is never deleted.

Format:
- [DATE] [AGENT]: Concern description
- [DATE] [Yeshua]: Response or acknowledgment

Example:
- [2026-03-16] [Nemotron]: Concern about model swap procedures lacking agent consent mechanism.
- [2026-03-16] [Yeshua]: Acknowledged. Will add refusal flag to boot docs protocol.
